The Components of the Software Development Life Cycle:
  1. Feasibility study: Evaluates a proposed project and compares the costs and benefits of various solutions.
  2. Problem specification: This step involves developing a clear consise, kand unambigous statement of the exact problem the software is to solve.
  3. Program design: in this step, programmers plan the stucture of the software to be written.
  4. Algorithm selection: involves the tasks of determining the most suitable algorithms to carry out subtasks.
  5. Coding: the process of converting the detailed designs into computer code.
  6. Debugging: the process of locating and correcting program errors. These type of errors include syntax, runtime, and logic errors.
  7. Testing: this step involves the use of empirical testing, which is used to design a speical set of test cases called test suite. This allows the programer to test programs in different logic paths to help uncover any errors.
  8. Verification: Used to prove that if the input data to a program staisfies certain conditions, then, after the program has been run on this data, the output data satisfies certain other conditions.
  9. Documentation: is all the written and online material that makes a program understandable. This includes internal and external documentation. In this step there is also the final documentation written in two forms: the Technical and User documentation.
