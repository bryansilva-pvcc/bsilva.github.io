Chapter 12: Models of Computation
1. What is the Purpose of a model?

A model is a simplified representation of a system to help enhance the understanding of it. Rather than the real thing, it can be used to train, study, and make informed decisions.

2. List four phenomena of a model:
  1. Captures the essence-the important properties-of the real thing
  2. Probably differs in scale from the real thing
  3. Omits some of the details of the real thing
  4. Lacks the full functionality of the real thing.
3. List four things a computing agent should be able to do:
  1. Accept input
  2. Store information in memory and retrieve it from memeory.
  3. Take actions according to algorithm instructions; the choice of what action to take may depend on the present state of the computing agent, as well as on the input item presently being processed.
  4. Produce output
4. List four characteristics fo an algorithm:
  1. Be a well-ordered collection
  2. Consist of unambiguous and effectively computable operations
  3. Halt in a finite amount of time
  4. Produce a result
5. What is the "halting problem" with regards to an uncomputable problem?

Decide, given any collection of Turning machine instructions together with any intial tape contents, whether that Turing machine will ever halt if started on that tape.

Chapter 13: Simulation & Modeling
Terms Definition
1. Scientific Method It is the method of observing the behavior of a system and formulating a hypothesis that tries to understand and explain that behavior.
2. Computational Model A model constructed usijng algorithmic procedures implemented as computer programs; also called simulation model.
3. Discrete Event Simulation A computational modeling technique that simulates the behavior of a system only at descrete points in time.
4. List and describe seven reasons to construct a simulation model.
  1. Existence: using a model can help with experimenting and the construction of the actual system.
  2. Physcial Realization: the system is not constructed from entities that can be represented by physical objects.
  3. Safety: experimenting on a replica can be safer then on the actaul system.
  4. Speed of construction: it might take too much time to construct a physical model and can sometimes be faster to design and build a computer simulation.
  5. Time Scale: some physical systems change too slowly or too quickly.
  6. Ethical behavior: some physical models have serious moral and ethical consequences, perhaps the best known being the use of animals in medical research.
  7. Ease of modification: if unsatisfied with a physical model, then a new one would need to be constucted. In a simulation, only need to change some numerical parameters and rerun the existing model.
