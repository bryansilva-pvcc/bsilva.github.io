Questions and Answers

  • Q. What is a version control system?
  • A. Also known as VCS, it is a system that records changes to a file or set of files overtime. This is extremely helpful beucase it allows other workng on a project to collaberate on it, see previous versions, and monitor changes made.
  • Q. What is Git?
  • A. Creatd by Linus Torvalds in 2005, Git is version control system (VSC) that allows users to track code changes, track who made changes, and codeing collaboration.
  • Q. What is GitHub?
  • A. It is a developer platform that allows developers to create, store, manage, and share their code to everyone. Currnelty I'm using GitHub as like a hosting service for others to view my current HTML assignemts online.
  • Q. What is the difference between Git and GitHub?
  • A. Though similar in name, they are not the same. GitHub makes tools that use Git. GitHub is a GUI as to Git is a software used in the command-line.
  • Q. Who started Github and how was it started?
  • A. Developed in October of 2007, it was started by Tom Preston-Werner, Chris Wanstrath, P.J. Hyett and Scott Chacon. They started it by using Ruby on Rails which is a sever-side web application framework.
  • Q. What company owns it now?
  • A. GitHub is currently owned by Mircosoft since 2018.
  • Q. How mucn does a GitHub account cost?

    • Enterprise: $21/ montly
  • Q. What is the Octocat?
  • A. The Octocat is GitHub's mascot created by graphic designer Simon Oxley.

Git / GitHub Terms:

  • Term: Respository
  • Definition: This is where you can store your code, files, and each file's revision history.
  • Term: Commit
  • Definition: This command captures a snapshot of the projects currently staged changes in the respository.
  • Term: Fork
  • Definition: Forks allows to copy another's users repository and also make changes without affecting the orginal repository.
  • Term: Push
  • Definition: means to send your committed changes to a remote respository on your GitHub account.
  • Term: Pull Requests
  • Definition: Are proposed changes to a respositoy submitted by another developer and is either accepted or rejected by the collaborators.
  • Term: Workflows
  • Definition: Located under the actions tab on your GitHub account, it is a collection of your GitHub actions that shows your running tests and deployments.
  • Term: Issues
  • Definition:These are suggested improvements, tasks or questions related to the respository.
  • Term: Raw Button
  • Definition: When clicked you can view or copy the raw content of a file without any formating or styling.
  • Term: Blame Button
  • Definition: When clickeed this feature allows you to view a line-by-line revision history and who made the change and when.
