Von Neumann computer architecture, 4 major subsystems:
  1. Memory: the functional unit fo a computer that stores and recieve instructions and data being executed.
  2. Input/Output: the device that allow a computer system to communicate and interact with the outside world and store information.
  3. Arithmetic/Logic Unit (ALU): performs mathematical and logical operations as addition, subtraction, and comparison for equality.
  4. Control Unit: fetches and executes instructions stored in the memory of the computer.

Von Neumann computer architecture, 3 characteristics:
  1. The four major substsystems that include: memory, input/output, the artihmetic/logic unit (ALU), and the control unit. The ALU and the control unit are often together and called the CPU.
  2. The stored program unit. This refers to the instructions to be executed by the computer are represented as binary values and stored in memory.
  3. The sequential execution of instructions. This is when one instruction at a time is fetched from memory to the control unit, where it is decoded and executed.

Computer Architecture Terms
  1. RAM: Random Access Memory: this is a memory structure in which each cell has an address and it takes the same amount of time to fetch or store any cell.
  2. Cache: A high-speed memory unit, that keeps a copy of memory cells with a high likelihood of access in the near future.
  3. Principles of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. When a computer uses something, it will probably use it again very soon.
    2. It will probably use the "neighbors" of this item very soon.
  4. ROM: Read-Only Memory: is a special type fo random access memory into which information has been prerecorded during manufracture.
  5. Memory Cell Size: This is the number of bits stored in a single memory cell.
  6. Memory Cell address: Is a unique numberic identifier assigned to each memory cell.
  7. Maximum Memory Size: Also known as address space, is the maximum amount of memory that a computer can physically hold and is deteremined by the size of the address field.
  8. Register: There are two registers that implenment the fetch and store operations, whose purpose is to hold two operands: MAR and MDR.
  9. Fetch: Searches the contents fo the memory cell with a specified address and return those contents as teh result of the operation.
  10. Store: meaning to store the specified value into the memroy cell specified by address.
  11. Memory Access Time: This is the timeit takes to fetch ro store the contents of a single memory cell.
  12. DASD: Direct Access Storage Devices: is a mass storage device in which each unit of information is associated with a unique address, but the time to access each piece of information may not be the same.
  13. Arithmetic and Logic Unit (ALU): A computer subsystem that performs mathematical and logical operations as addition, subtraction, and comparison for equality.
  14. Control Unit: A coputer subsystem that fetches and executes instructions stored in the memory of the computer.

Chapter 6: An Introduction to System Software and Virtual Machines

Four things the interface between the computer and the hardware does:
  1. Hides from the user the messy details of the underlying harware.
  2. Presents information about what is happening in a way that does not requre in-depth knowledge of teh internal structure of the system.
  3. Allows easy user access to the resources available on this computer.
  4. Prevents accidental or intentional damage to hardware, programs, and data.

Responsibilities of systems software:
  1. Hides the complex and unimportant(to the user) details of the internal structure of the Von Neumann architecture.
  2. Presents important information to the user in a way that is easy to understand.
  3. Allows the user to access machine resources in a simple and efficient way.
  4. Provides a secure and safe environment in which to ooperate.

Terms
  1. Systmes Software: A collection fo computer programs that manage the resources of a computer and facilitate access to those resources.
  2. User Interface: An intuitive visual overview as well as graphical contol of the computer's capabilities and services.
  3. Language Services: Programs known as assemblers, compliers, and interpreters which allow programs to be written in high-level, user-oriented language instead of machine language.
  4. Memory Managers: Allocate memory space for programs and data and retrieve this memory space when it is no longer needed.
  5. Information Managers: These programs handle the organization, storage, and retrieval of information on storage devices such as the hardrive, DVDs, flash drives, and tapes.
  6. I/O Managers: Software packages that allow the user to easily and efficiently use the many different types of input and output devices that exist on a modern computer system.
  7. Scheduler: Keeps a list of programs ready to run on the processor, and it selects the one tha will execute next. It also allows you to have several different programs active at a single time.
  8. Utilities: Library routines that provide a wide range of useful services either to a user or to other system routines. Sometimes these collections can be called program libraries.
  9. Assembly Language: A low-level programming language that maps 1:1 into machine language.

Functions of an Operating System
  1. User interface management
  2. Control of access to teh system and to data files
  3. Program scheduling and activation
  4. Efficient resouce allocation
  5. Deadlock dectection and error dectection.
