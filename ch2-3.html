Chapter 2: Briefly define:
  1. Natural Language:
    The language in which we speak in our everyday lives.
  2. Pseudocode:
    This is used to design and represnt alogorithms. It is a form of notation that looks like high-level programming language but is written in everyday or natural language.
  3. Sequential Operation (straigt-line algorithm):
    Also known as sequential algorithm, this term means to exucute instructions in a straigt-line from top to bottom and then stops.
  4. Input:
    Data values from the outside world that can be used in later instructions
  5. Outpout:
    Takes the input values and returns them, sends results from the compting agent to the outside world.
  6. Conditional Statement:
    This is the "question-asking" operatoins of an algorithm. This algorithm executes the next operation based on if set condition is met.
  7. Boolean Condition:
    This executes code based on if the statments true or false. This is a decision making process that excutes a specific sequence based on true or false.
  8. Algorithmic Discovery Process:
    This is the process of finding an algorithmic solution to a given problem, often teh most challenging and creative part of the problem-solving process.

Chapter 3: Briefly describe or define:
  1. Why algorithms should be correct.
    The intended purpose of an algorithm is to solve a problem and give a result. If the problem is incorrect or the result is wrong it fails it's purpose.
  2. Why algorithms should be efficient.
    Efficiency refers to a program's time (speed in running the algorithm on a real compuer) and space (the programs memory useage)
  3. Program Maintence
    Maintence accurs to both fix any errors that may have come from repeated usage and to extend a program to meet any new requirements.
  4. Importance of algorithm: ease of understanding.
    A lot of time and money is put into most programs. Often times, it is not the person who wrote the orginal program is incharge of maintaining it. Therefor, it is important to create the program to where it is easy to understand.
  5. Algorithmic Elegance
    Elegance is the algorithmic equivalent of style. Making your algorithms appear clear and simple.
  6. Sequential Search Algorithm
    An algorithm that searches for a target value in a random list by checking each list item one by one until target value is found or has hit the end of the list.
  7. Order of Magnitude
    The concept of which the an algorithm whose work grows or varies as costant times as the input size. Order of magnitude is also used to classify algorithms
  8. Selection Sort
    This algorithm performs a task that can sort numbers in ascendnig order, in a list it wll keep moving larger items toward the back of the list.
  9. Flop (floating point operations)
    The operation in which a computer perfroms additions or subtractions of real numbers.
  10. Teraflop
    A unit of computing speed that means a trillion floating-point operations per second.
  11. Petaflop
    Also a unit of computing speed tahat mean quadrillion floating-point operations per second.
  12. Parallel Processor
    A computing system with multiple core prcessors and that can performs flops much faster.
  13. Data Cleanup Problem
    Algorithms in which are used to detect and remove incomplete or unnecessary data from a program.
  14. Binary Search Algorithm
    This algorithm searches for a target value in a sorted list by checking at teh midpoint and then repeatedly cutting the search section in half.
  15. Pattern Matching
    A pattern-matching algorithm that does a "forward march" through a set of text and attemps to match each pattern character against the text characters.
